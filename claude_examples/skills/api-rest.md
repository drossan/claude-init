# Skill: Arquitectura de API REST (api-rest)

## Propósito
Diseñar y exponer servicios web siguiendo los principios y restricciones de la arquitectura RESTful.

## Responsabilidades
- Utilizar correctamente los verbos HTTP (`GET`, `POST`, `PUT`, `DELETE`, `PATCH`) según la acción.
- Diseñar URIs intuitivas y orientadas a recursos (sustantivos en plural).
- Implementar códigos de estado HTTP semánticos (200 OK, 201 Created, 400 Bad Request, 404 Not Found, 500 Internal Server Error).
- Gestionar la negociación de contenido y asegurar respuestas JSON consistentes.
- Implementar paginación, filtrado y ordenación en endpoints de listado.
- Asegurar la statelessness (ausencia de estado) en las peticiones de la API.
